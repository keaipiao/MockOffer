# 部署手册

> 项目级冷启动手册——**从零到当前版本**，怎么把项目完整跑起来。
> living document：每次上线引入新环境变量 / 服务 / 步骤，都累积进来，让它**永远停在当前 HEAD**。
> 单次上线的快照在各功能 `发布部署.md`；这里是「从 0 到现在」的完整版。
> 章节固定、按编号保留，不适用就注明「不适用」、不删。

---

## 1. 前置准备

- **部署（容器化）**：Docker + Docker Compose。
- **本地开发**：JDK 21、Node.js 20+（前端 Next 16）、Docker（起 PG / Redis 依赖与跑测试用的 Testcontainers）。
- **第三方服务**：
  - GitHub OAuth App（用 GitHub 登录则需要，申请见 `docs/功能/01-账号体系/发布部署.md` §2.3）。
  - 腾讯云 SES（prod 真实发验证码邮件需要；dev 用日志代发，免申请）。

---

## 2. 获取代码与配置

1. clone 仓库。
2. 复制配置样板并填真实值：`cp deploy/.env.example deploy/.env`，填 DB 密码、`JWT_SECRET`（≥32 字节）、GitHub 凭证、SES 凭证。
3. dev / prod 用各自的配置区分（`SPRING_PROFILES_ACTIVE`，详见 `架构总览.md` §6）：
   - **能用环境变量区分 dev/prod 的差异**（DB/Redis 主机、`cookie.secure`、前端地址）写在 `application-{dev,prod}.properties`，**不进 `.env`**。
   - `.env` 只放凭证 / 密钥等「环境无关但敏感」的值。

---

## 3. 环境变量汇总

> 全项目环境变量的**权威清单**——各功能上线新增的 env 都累积到这里（与各功能 `发布部署.md` 同步）。
> 遵守配置最小化：只列真正必要的（见 `CLAUDE.md`「编码红线 · 配置最小化」）。

| 变量 | 必填 | 默认 | 说明 |
|---|---|---|---|
| `POSTGRES_USER` / `POSTGRES_PASSWORD` / `POSTGRES_DB` | 是 | mockoffer | PostgreSQL 凭证与库名（prod 改强密码） |
| `JWT_SECRET` | 是 | dev 占位串 | JWT HS256 密钥，≥32 字节，prod 必换 |
| `GITHUB_CLIENT_ID` / `GITHUB_CLIENT_SECRET` | 用 GitHub 登录则必填 | dev-placeholder | GitHub OAuth 凭证 |
| `TENCENT_SES_SECRET_ID` / `TENCENT_SES_SECRET_KEY` | prod 发码必填 | 空 | 腾讯云 SES 凭证 |
| `SES_FROM_EMAIL` / `SES_TEMPLATE_ID` | prod 发码必填 | 空 | 发件邮箱与审核通过的模板 ID |
| `SPRING_PROFILES_ACTIVE` | prod 必填 | dev | 环境 profile，compose 设 `prod` |
| `BACKEND_INTERNAL_URL` | 容器部署必填 | http://localhost:8080 | 前端 SSR server→后端 内网地址，compose 设 `http://backend:8080`（浏览器侧请求仍走公网地址） |
| `SPRINGDOTENV_IGNORE_IF_MISSING` | 容器部署必填 | — | 容器内无 `.env` 文件时不报错，compose 设 `true` |
| `MINIO_ROOT_USER` / `MINIO_ROOT_PASSWORD` | 否 | minioadmin | 对象存储（后续里程碑用） |

## 4. 启动步骤（从零）

**方式 A · 容器化（推荐，一键起全套）**

```bash
cd deploy && docker compose up -d --build
```

起 PostgreSQL / Redis / MinIO / 后端 / 前端；后端启动时 Flyway 自动建表。

**方式 B · 本地开发（前后端分别起）**

```bash
# 1. 依赖：只起 PG / Redis（compose 局部）
cd deploy && docker compose up -d postgres redis
# 2. 后端（dev profile，读 deploy/.env）
cd backend && ./mvnw spring-boot:run
# 3. 前端
cd frontend && npm install && npm run dev
```

## 5. 验证

- 后端健康：`curl http://localhost:8080/api/health` → `200`。
- 前端：浏览器开 `http://localhost:3000`，未登录见落地页；邮箱验证码登录（dev 验证码打在后端日志 / 从 Redis `otp:code:{email}` 取）或 GitHub 登录跑通；设置页绑定 / 解绑正常。

---

## 6. 运维常用操作

- **更新部署**：拉新代码 → `docker compose up -d --build` → 重启。
- **看日志 / 排障**：`docker compose logs -f backend`（或 frontend / postgres / redis）。
- **备份**：PostgreSQL 数据在具名卷 `pgdata`（挂载点 `/var/lib/postgresql`，注意 PG18 路径，见 `踩坑记录.md`）；定期 `pg_dump`。
- **回滚**：单次上线的回滚步骤见对应功能 `发布部署.md` §4。
