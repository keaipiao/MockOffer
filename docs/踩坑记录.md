## 2026-06-14 · PostgreSQL 18+ Docker 镜像数据卷路径变更

- 现象：docker compose 起 `postgres:18-alpine` 容器立即 Exited(1)，日志「in 18+, these Docker images are configured to store database data in a 子目录…」，依赖它（`condition: service_healthy`）的后端连带起不来。
- 根因：PostgreSQL 18 官方镜像把 PGDATA 移到 `/var/lib/postgresql` 下的子目录、VOLUME 声明从 `/var/lib/postgresql/data` 改为 `/var/lib/postgresql`；仍按旧习惯把具名卷挂到 `/var/lib/postgresql/data` 会被镜像主动拒绝启动（防数据混淆/丢失）。
- 正确做法：`postgres:18+` 的数据卷挂到 `/var/lib/postgresql`（不是 `/data`）。
- 适用范围：任何用 postgres:18 及以上官方镜像 + 具名卷的 compose / k8s 配置。
