## 1. 测试计划

### 1.1 单元测试

纯 Mockito、不依赖 DB / Redis。

| 模块 | 测试点 | 覆盖 |
|---|---|---|
| 账号归一与登录方式 | 命中身份进同账号 / 未命中建新号 / 账号禁用拒绝；绑定占用拒绝与幂等；解绑「至少保留一种」；换绑邮箱占用拒绝与新增 | 正常流程 / 异常分支 / 边界值 |
| 验证码 | 发码间隔限流、每日次数上限、邮箱格式校验、发送失败回退；校验码正确/错误/过期/失败锁定/一次性消费 | 正常流程 / 异常分支 / 边界值 |
| 令牌与会话 | access 真签真验 round-trip、jti 唯一；refresh 一次性消费（轮换 + 重放拒绝）、access 黑名单即时吊销 | 正常流程 / 异常分支 |

### 1.2 集成测试

Testcontainers 真起 PostgreSQL + Redis，走完整 HTTP 链路（基类 `IntegrationTestBase` 单例容器、全套件共享、自包含不依赖本地服务）。

| 场景 | 验证目标 |
|---|---|
| 邮箱登录金路径 | 发码 → 登录 → 当前用户 `/me` → 登出 → 同一 access 被吊销返回 401 |
| 权限边界 | 未登录访问 `/api/auth/me`、`/api/account/identities` 返回 401 |
| 上下文加载 | 全量 Spring 上下文（含 Security / OAuth2 / Flyway 迁移）能正常启动 |

### 1.3 端到端 / 联调（金路径 + 边界）

**金路径**（浏览器实测，连本机 Chrome）：
1. 未登录访问首页 → 落地页（导航「首页」+「登录」）
2. 邮箱登录：发码 → 6 格验证码 → 落地设置页，邮箱行显示地址
3. GitHub OAuth 登录 → 落地工作台（「工作台」胶囊高亮），设置页 GitHub 行显示 `@用户名`
4. 绑定邮箱（一步式弹窗）→ 解绑，列表即时刷新

**边界场景**（逐个验证不崩、给出合理反馈）：
- 验证码错误 / 过期 → 顶部红色 toast 提示，5s 后带退出动画消失
- 解绑唯一登录方式 → 被「至少保留一种登录方式」拒绝（自定义确认弹窗）
- 未登录 / 越权访问受保护接口 → 401（前端落地页 / 跳登录）
- SSR 登录态：硬刷新首页/设置页不闪、不误跳登录

### 2.3 发现并修复的 Bug

| 现象 | 根因 | 修复 | 防御措施 |
|---|---|---|---|
| 设置页 GitHub 行不显示用户名 | `/me` 与 `/account/identities` 各写一套身份标识逻辑，前者把 github 写死 null | 标识逻辑下沉 `AccountService.accountLabel` 一处共用 | 单一数据源，杜绝双份逻辑分叉 |
| 未登录调 `/me` 返回 500 | `/api/auth/me` 落在 `/api/auth/**` 的 permitAll，principal=null 触发 NPE | 把 `me` 移出 permitAll（authenticated），未登录正常 401 | 集成测试覆盖「未登录访问受保护接口 401」 |
| GitHub 登录偶发一直转圈 | 回调内同步调 `/user/emails`（RestClient 无超时）拖慢整条回调 | 去掉该调用，GitHub 改展示 `@用户名`，scope 收成 `read:user` | 不在回调链路放无超时外部调用 |

## 3. 不测范围与理由

> 明确「不测什么、为什么」——这是有意识的取舍，不是遗漏。

- **前端不写自动化测试**：M1 范围内前端无测试栈，改用浏览器逐项手动 QA（见 §1.3、看板节点 #2573 笔记）；后续若前端逻辑变重再引入。
- **腾讯云 SES 真实发信不测**：dev 用 `LogEmailSender` 日志代发，验证码直接从 Redis 取；真实发信放到生产冒烟测试覆盖。
- **GitHub OAuth 与 GitHub 服务器交互不做自动化**：依赖外部服务与真实账号会话，靠浏览器联调 + 生产冒烟覆盖。
